## DWPose 输出无法给whk_3DPose_edit_2D_DocKr 使用
报错：
```
File "C:\Users\home\Documents\ComfyUI\custom_nodes\ComfyUI-3D-OpenPose-Editor2026\nodes.py", line 334, in render_dw_pose
   polygon = cv2.ellipse2Poly((int(mY), int(mX)), (int(length / 2), scaled_stickwidth), int(angle), 0, 360, 1)
                                                    ^^^^^^^^^^^^^^^
ValueError: cannot convert float NaN to integer
```


C:\Users\home\Documents\ComfyUI\custom_nodes\ComfyUI-3D-OpenPose-Editor2026\nodes.py - line334:

因为 numpy 整数溢出。
X[0] - X[1] = 292765 - 232114 = 60651
(60651)² = 3,678,542,601

numpy 的 int32 最大值是 2,147,483,647（约21亿），而 60651² = 36.8亿 > 21亿 → 溢出变成负数 → 两个平方相加后还是负数 → np.sqrt(负数) = NaN

根本原因： DWPose 输出的坐标不是像素值（0-512），而是某种未归一化的大整数，3D Editor 直接拿来算就溢出了。

这确认了格式不兼容——3D Editor 期望坐标是正常像素范围，但收到的是几十万级别的原始数据。

所以结论还是：断开连线，手动摆姿势，或者直接用 DWPose 的骨骼图跳过 3D Edit
