# 人脸图像质量评估：模型与方法

> 适用场景：AI影视前期制作，用于筛选适合换脸/角色一致性的模特参考照片
> 整理日期：2026-05-04

---

## 一、优质参考照片标准

### 必须满足
- 正脸或接近正脸（偏角不超过45度）
- 面部清晰，无运动模糊
- 光线均匀，五官不过曝/过暗
- 分辨率至少 512×512，越高越好
- 无大面积遮挡（口罩、墨镜、刘海遮眼）

### 加分项
- 提供4张以上，包含不同角度（正面、左45°、右45°）
- 不同光线条件（自然光、室内光）
- 表情自然，不夸张

### 会导致失败的情况
- 侧脸超过60度
- 强逆光/阴影遮脸
- 美颜过度（磨皮严重，五官特征被抹掉）
- 小图、低像素（人脸区域低于512px）

## 二、人脸相关模型谱系

### 1. 检测类（找到脸在哪）

| 模型 | 特点 | 适用场景 |
|------|------|---------|
| **RetinaFace** | InsightFace 出品，业界标杆，检测精度极高 | 精品筛选 |
| MTCNN | 老牌，速度快但精度略差 | 快速初筛 |
| YOLOv8-face | 速度极快，适合实时场景 | 批量处理 |

> 注：RetinaFace 是 InsightFace 家族的一部分，不是独立体系

### 2. 关键点/姿态估计（测偏角）

| 模型 | 特点 | 输出 |
|------|------|------|
| **MediaPipe Face Mesh** | Google 出品，478个关键点 | yaw/pitch/roll 精确偏角 |
| 3DDFA | 专门做3D人脸姿态 | 比 MediaPipe 更准 |

### 3. 质量评估（这张脸够不够好）

| 模型 | 特点 | 推荐指数 |
|------|------|---------|
| **CR-FIQA** | 最新人脸质量评估，输出0-1分 | ⭐⭐⭐⭐⭐ |
| SER-FIQ / FaceQnet | 专门做质量评分的早期方案 | ⭐⭐⭐ |
| **MagFace** | 特征质量评估，越清晰/越正脸分越高 | ⭐⭐⭐⭐ |
| **AdaFace** | 低质量图片下更稳，适合评估模糊照 | ⭐⭐⭐⭐ |

### 4. 特征提取/识别类（这张脸是谁）

| 模型 | 特点 |
|------|------|
| **ArcFace / CosFace** | InsightFace 核心，人脸识别标准方案 |
| AdaFace | 低质量场景下识别更稳定 |

## 三、推荐的自动化评估流程

适合需要批量筛选（几百张以上）的场景：

```
步骤1: MediaPipe Face Mesh
       → 测偏角（yaw > 45° 直接过滤）

步骤2: CR-FIQA
       → 质量打分（score < 0.5 标记为低质）

步骤3: InsightFace (ArcFace)
       → 最终验证能否提取到稳定特征向量

输出: 每张照片的评分报告 + 推荐/不推荐标签
```

**实现方式：** Python 脚本，依赖 insightface、mediapipe、onnxruntime

## 四、简单场景的替代方案

### 数量少（几十张）
直接发给 Claude/GPT-4V 进行视觉分析，可判断：
- 角度、光线、清晰度
- 是否适合作为 InstantID/换脸参考图
- 给出具体改进建议

### 实战验证
上传到 liblib.art 跑一次 InstantID，看输出质量 —— 最直接的判断方式

## 五、关键结论

> 对于 InstantID / 换脸 / 角色一致性来说：
> **正脸 + 清晰 + 无过度美颜** 是三个最核心的要求
> 
> 工具链: MediaPipe（角度）+ CR-FIQA（质量）+ InsightFace（验证）
> 简单场景直接肉眼+视觉AI判断即可

## 六、用 LLM 视觉分析评估照片质量

当照片数量不多（几十张以内），可以直接发给 Claude / GPT-4V 等视觉模型进行判断。以下是 LLM 评估时的检查维度：

### 检查维度

**1. 人脸几何信息够不够**

AI 换脸的本质是"提取人脸特征向量然后迁移"，因此重点看：
- 脸有没有被裁掉（额头/下巴/脸颊）
- 偏角多少（侧脸越多，特征丢失越多）
- 五官是否都清晰可见

**2. 信噪比**
- 有没有运动模糊、压缩噪点、强烈阴影
- 美颜/磨皮是否过度（磨皮会抹掉毛孔纹理，导致 AI "认不出"真实皮肤特征）

**3. 光线分布**
- 顺光最好，侧光次之，逆光很差
- 过曝区域（白脸）= 信息缺失
- 强阴影区域 = 信息缺失

**4. 分辨率与细节**
- 放大后眼睛虹膜、睫毛是否清晰
- 低于 512px 的人脸区域效果会明显变差

### LLM 判断的本质

LLM 的判断是**经验+逻辑推理，不是跑算法**。

基于 InsightFace / InstantID 等模型的技术论文和大量用户反馈，知道"什么情况会导致失败"，然后通过视觉分析预判。不会输出精确评分，但能给出定性判断，例如：

- "这张能用，但偏角稍大，建议补一张正面"
- "这张美颜太重，模型可能识别不稳定"
- "这组很好，角度覆盖全，直接用"

### 适用场景对比

| 场景 | 推荐方法 |
|------|---------|
| 照片 < 50张 | 直接发给 LLM 视觉分析，最快 |
| 照片 50-500张 | Python 脚本自动化（MediaPipe + CR-FIQA） |
| 照片 > 500张 / 视频提帧 | 全自动流水线（见下节） |

## 七、从视频中提取优质人脸帧

适合场景：有一段几十分钟的视频素材，需要批量筛选适合 LoRA 训练或 InstantID 的关键帧。

### 自动化流程

```
输入视频（如40min）
  → ffmpeg 按帧提取（每秒1帧，约2400张）
  → InsightFace 人脸检测（过滤无脸/多脸帧）
  → MediaPipe 偏角过滤（yaw > 40° 丢弃）
  → Laplacian 方差清晰度评分
  → 亮度均匀性评分
  → 综合排序，输出 Top N 帧
  → 生成 CSV 报告（帧号、时间码、各项评分）
```

### Python 脚本依赖

```bash
pip install insightface opencv-python mediapipe onnxruntime
# M2 Mac 可正常运行，无需独立显卡
```

### 脚本功能

- 输入：视频文件路径 + 输出目录
- 自动按帧提取（可配置采样率）
- 人脸检测 + 偏角过大过滤
- 清晰度（Laplacian方差）+ 亮度均匀性评分
- 综合打分，输出推荐帧到指定目录
- 生成 CSV 报告：帧号、时间码、清晰度分、偏角、综合分
