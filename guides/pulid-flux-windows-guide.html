🪤 踩坑记录（按踩坑顺序）

坑1：T5-XXL 文本编码器加载失败
RuntimeError: mat1 and mat2 shapes cannot be multiplied (256x4096 and 10240x4096)
💡 原因：ModelScope 的 FLUX.1-dev 仓库里 t5xxl 是分片文件（model-00001-of-00002），ComfyUI 会把 clip_l 错误地加载进 t5xxl 的槽位，导致维度不匹配。
✅ 解决：从 HuggingFace comfyanonymous/flux_text_encoders 下载单文件版 t5xxl_fp16.safetensors，不要用 ModelScope 的分片版。
坑2：InsightFace 联网验证，本地文件形同虚设
requests.exceptions.ConnectTimeout: HTTPSConnectionPool(host='github.com', port=443)
💡 原因：InsightFace 每次加载模型都要先联网到 GitHub 验证完整性，国内连不上就直接失败，哪怕本地文件完好。
✅ 解决：把模型文件直接复制到 InsightFace 默认缓存目录（不是 junction）：
C:\Users\home\.insightface\models\buffalo_l\
C:\Users\home\.insightface\models\antelopev2\
或修改 storage.py 添加本地目录检测跳过下载。
坑3：EVA-CLIP 路径认知错误
HTTP Request: GET https://huggingface.co/QuanSun/EVA-CLIP/... （明明本地有文件还在下载）
💡 原因：PuLID 的 EVA-CLIP 加载器用 folder_paths.get_full_path("text_encoders", ...) 查找，找不到再去 models/clip，不会从 insightface 目录读。放错目录就会触发网络下载。
✅ 解决：把 EVA02_CLIP_L_336_psz14_s6B.pt 放到：
C:\Users\home\Documents\ComfyUI\models\clip\EVA02_CLIP_L_336_psz14_s6B.pt
坑4：facexlib 模型路径有坑
FileNotFoundError: 'C:\Users\home\Documents\ComfyUI\models\facexlib\parsing_bisenet.pth'
💡 原因：facexlib 会把检测/分割模型下载到 save_dir（由调用方传入）。PuLID 传的是 models/facexlib 目录，所以要手动把文件放进去，不能放到其他地方。
需要两个文件：detection_Resnet50_Final.pth（人脸定位）和 parsing_bisenet.pth（人脸分割）。
✅ 解决：手动下载两个文件，放到：
C:\Users\home\Documents\ComfyUI\models\facexlib\
坑5：PuLID v1.1.4 与新版 ComfyUI 不兼容（latent_shapes）
TypeError: pulid_outer_sample_wrappers_with_override() got an unexpected keyword argument 'latent_shapes'
💡 原因：ComfyUI 新版采样器加了 latent_shapes 参数，但 PuLID 插件（最后更新 2025-02-19）函数签名没有接收该参数。
✅ 解决：修改 custom_nodes\comfyui_pulid_flux_ll\pulidflux.py，在函数定义末尾加 **kwargs： 
# 改前
def pulid_outer_sample_wrappers_with_override(wrapper_executor, noise, latent_image,
    sampler, sigmas, denoise_mask=None, callback=None, disable_pbar=False, seed=None):

# 改后
def pulid_outer_sample_wrappers_with_override(wrapper_executor, noise, latent_image,
    sampler, sigmas, denoise_mask=None, callback=None, disable_pbar=False, seed=None, **kwargs):
坑6：PuLID v1.1.4 与新版 ComfyUI 不兼容（timestep_zero_index）
TypeError: pulid_forward_orig() got an unexpected keyword argument 'timestep_zero_index'
💡 原因：同上，ComfyUI 新版 flux/model.py 调用 forward 时加了 timestep_zero_index 参数，PuLID 的 forward 函数签名没有接收。
✅ 解决：同文件，找到 def pulid_forward_orig，在 attn_mask: Tensor = None, 后面加一行 **kwargs,： 
def pulid_forward_orig(
    self,
    img: Tensor,
    img_ids: Tensor,
    txt: Tensor,
    txt_ids: Tensor,
    timesteps: Tensor,
    y: Tensor,
    guidance: Tensor = None,
    control = None,
    transformer_options={},
    attn_mask: Tensor = None,
    **kwargs,          # ← 加这一行
) -> Tensor:
